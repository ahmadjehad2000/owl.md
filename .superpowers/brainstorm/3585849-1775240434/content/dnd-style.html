Drag-and-drop — what should it do?

The DB already has a parent_id column — how much of it do we use?

A

Reorder only

Drag notes up/down to change their order in the list. No nesting — all notes stay flat. Simple to implement and use.

B

Nest into folders (full hierarchy)

Drag a note onto another note or folder to make it a child. The sidebar becomes a tree. Uses parent_id fully. More powerful but more complex.

C

Drag into named folders only

Notes can be dragged into folder groups (like "Research", "Daily"). Folders exist as labels, not as notes themselves. Middle ground.
