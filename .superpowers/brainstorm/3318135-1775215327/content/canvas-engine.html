Canvas / Whiteboard Engine

Infinite pan/zoom, draggable note cards, connectors, grouping. Which foundation?

A

React Flow

Node-and-edge graph canvas, React-native, production-proven

Pros

  • Nodes are React components — you render full TipTap editors inside canvas cards
  • Built-in pan/zoom, minimap, edge routing, selection, grouping
  • Used by Obsidian Canvas, many production tools
  • Excellent TypeScript support, well-maintained
  • Custom node types trivial — note card, image card, mind map card all just components

Cons

  • Pro license required for some advanced features (background, grid snap)
  • Large canvas with hundreds of nodes needs viewport culling (manageable)
B

tldraw

Full whiteboard SDK — shapes, freehand, sticky notes, arrows

Pros

  • More freeform / whiteboard-native feel (sticky notes, freehand drawing)
  • Rich shape library built in
  • Custom shapes can embed React content
  • Good persistence model

Cons

  • Heavier runtime — designed for the full whiteboard experience
  • Custom shape / embed integration is more complex than React Flow nodes
  • Less node-graph-oriented — connecting shapes is not its primary model
  • License changed to BSL 1.1 — commercial use requires license
C

Custom (CSS transform + pointer events)

Build your own lightweight infinite canvas over React

Pros

  • Zero dependency, full control
  • Tailor exactly to owl.md's data model
  • No licensing concerns

Cons

  • Pan/zoom, snap-to-grid, edge routing, selection box — all built by hand
  • Significant scope for phase 3
  • Not recommended unless you want the canvas to be a differentiator
My recommendation

React Flow. Its node-as-React-component model means you can drop a live TipTap editor directly inside a canvas card — exactly what your spec asks for. Obsidian Canvas uses this same approach. The edge routing, minimap, grouping, and selection are all production-ready. tldraw is excellent but more whiteboard-focused than knowledge-graph-focused.
