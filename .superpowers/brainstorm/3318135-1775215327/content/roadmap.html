Phased Roadmap

Five phases, each delivering a fully usable vertical slice. Phase 1 is the usable core — everything after extends it without breaking it.

✦ MVP = Phase 1 only — vault, editor, backlinks, search. Fully usable as a note-taking app before Phase 2 begins.
1
Phase 1 Core Local Vault & Editor Unlock: usable note-taking app
Vault create / open Markdown CRUD TipTap editor Folder tree sidebar [[WikiLink]] extension Backlink index Backlinks panel chokidar FS watcher SQLite + migrations FTS5 full-text search Search modal Aurora glass UI Pinned / recent notes Autosave electron-vite scaffold
Foundation that all later phases extend. The .owl/ directory design and IPC bridge are established here and never restructured.
2
Phase 2 Rich UX & Convenience Unlock: power-user workflow
Command palette ⌘K Tabs + split panes Slash commands Callout blocks Outline sidebar Templates Daily notes Tags + saved searches Drag-and-drop hierarchy Properties panel Graph view (React Flow) Note metadata YAML Keyboard shortcut map
Everything that makes the editor feel native and fast. Graph view is introduced here using React Flow before the full canvas is added in P3.
3
Phase 3 Whiteboard & Mind Maps Unlock: spatial knowledge
React Flow canvas Infinite pan / zoom Note cards on canvas Live note preview cards Image / file / text cards Labeled connectors Node grouping Canvas persistence (SQLite) Mind map editor Note → mind map Mind map → note hierarchy Node ↔ note linking Viewport state persistence
Canvas and mind map reuse the same React Flow engine. Note cards on canvas embed live TipTap editors — this is why TipTap was chosen in P1.
4
Phase 4 Ollama AI Integration Unlock: grounded local LLM
Ollama connection config Model selector nomic-embed embeddings usearch HNSW index Note chunking pipeline RAG context assembly Ask current note Ask linked notes Ask folder / workspace Ask canvas selection Ask mind map Citation panel Summarize / rewrite / extract Suggest backlinks Incremental embed invalidation
AI is purely additive — all of P1–P3 works with Ollama unreachable. The AI panel shows a "Ollama not connected" state gracefully.
5
Phase 5 Backup, Hardening & Polish Unlock: production-ready
Google Drive OAuth Snapshot bundling Versioned restore UI Conflict detection Scheduled backups Crash recovery journal Corrupted index repair Large vault perf tuning Import from Obsidian Import from Notion Export to PDF / HTML Plugin extension points
Hardening, import/export, and backup. The plugin extension points are API-only in v1 — no plugin sandbox yet, just documented hooks for future work.
