Desktop Framework Choice

This is the single most foundational decision — it shapes build tooling, native access, bundle size, and long-term maintainability.

A

Electron

Node.js + Chromium, full JavaScript stack end-to-end

Pros

  • Huge ecosystem (VS Code, Obsidian, Notion desktop all use it)
  • Full Node.js APIs — filesystem, native modules, sqlite3, everything
  • One language (TypeScript) across the entire app
  • Best-in-class native module support (better-sqlite3, chokidar, etc.)
  • Proven at scale for knowledge apps
  • Simpler mental model — renderer ↔ main process IPC

Cons

  • Large binary (~150–200 MB with app)
  • Higher memory baseline (~200 MB idle)
  • Slower cold start vs native
B

Tauri 2.0

Rust backend + system WebView, React/TS frontend via IPC

Pros

  • Much smaller binary (~10–30 MB)
  • Lower memory footprint (~50–80 MB idle)
  • Rust backend is fast and memory-safe
  • Native filesystem watcher via Rust (no chokidar)
  • Better sandboxing model long-term

Cons

  • Rust required for backend — split language stack (Rust + TS)
  • System WebView inconsistencies (especially Windows)
  • SQLite via Tauri plugin is less mature than better-sqlite3
  • More IPC ceremony for complex backend ops
  • Ecosystem for knowledge app primitives is thinner
My recommendation

Electron. Obsidian and VS Code both prove Electron works for complex local-first knowledge tools at production scale. The better-sqlite3 driver, chokidar FS watcher, and Node native module ecosystem are all battle-tested here. Tauri's advantages (binary size, memory) matter more for simple utilities than for a feature-dense workspace app. The Rust split-stack cost is real and will slow iteration during a phase-1 build.
