Editor Engine

Your spec calls for "hybrid markdown + block model" with slash commands, drag-and-drop blocks, and live preview. Both options can do this — the trade-off is in how much you build vs how much you get for free.

A

ProseMirror (via TipTap)

Rich document model — thinks in nodes and marks, not raw text

Pros

  • TipTap gives you slash commands, drag handles, block types out of the box
  • Native block model — block IDs, block references, drag-and-drop reordering are natural
  • Notion, Linear, and many block editors use this exact stack
  • Collaborative editing ready (Y.js integration is first-class)
  • Richer extension ecosystem for callouts, tables, embeds

Cons

  • Document model is JSON, not raw Markdown — need serialization layer to stay markdown-on-disk
  • Markdown round-trip requires careful mapping (especially for custom nodes)
  • Heavier than CodeMirror for simple text editing
B

CodeMirror 6

Text-first editor — thinks in characters and ranges, with a powerful extension system

Pros

  • Raw markdown stays as the document model — zero serialization risk
  • Live preview via decorations (Obsidian's approach)
  • Extremely fast even in large documents
  • Lightweight and predictable
  • Perfect markdown fidelity — what you see on disk is what the editor edits

Cons

  • Block IDs and block-level drag-and-drop must be built manually
  • Slash commands, callout types, embed blocks require custom extensions
  • More boilerplate to get to feature parity with TipTap
My recommendation

ProseMirror via TipTap. Your spec explicitly wants slash commands, drag-and-drop block reordering, block IDs for AI chunking, and callout/embed blocks. TipTap gives you all of that in v1 without building it from scratch. The markdown serialization cost is manageable — TipTap has a solid @tiptap/extension-markdown serializer, and notes stay human-readable on disk. CodeMirror is the right choice if you want Obsidian-exact markdown fidelity with simpler editing — but you'd be building the block layer yourself.
